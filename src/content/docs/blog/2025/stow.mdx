## 前言

工欲善其事，必先利其器  

每次拿到新設備第一件事一定就是花不少時間設定自己的開發工具比如說 tmux， nvim   

往往這些工具都需要設定檔，你可以把這些設定檔集中在一個 repo 裡面管理  

但會有一個問題是你需要自己把這些設定檔弄到指定的路徑  

像是 tmux 的 config 要放在 `~/.tmux.conf` 下，nvim 需要放在 `~/.config/nvim` 下面  

這件事蠻麻煩的，今天介紹的工具 **stow** 就在解決這個問題


## 快速上手

### 下載

[stow](https://www.gnu.org/software/stow/) 是 GNU 出的 symlink manager   

透過 symlink 來統一維護，，但又做到放在各個地方   

第一步當然是先下載

<Tabs>
  <TabItem label="Linux (Debian)">
```sh
sudo apt install -y stow
```
  </TabItem>
  <TabItem label="MacOS">
```sh
brew install stow
```
  </TabItem>
</Tabs>

### 專案架構

先創建一個新的 directory 來存放這些設定檔

```sh
cd ~ && mkdir dotfiles
```


接著就可以依據類別創建對應的資料夾  

像是我可以創建資料夾來專門存放 vim 的設定檔，名字不一定要叫 vim，你自己能辨認即可  

接著就在對應的資料夾內依照正常方式擺放你的設定檔

不太好懂，直接看例子，左邊是 stow 會為你創建的 symlink，右邊是你在專案中擺放的檔案

1.   `~/.vimrc` -> `~/dotfiles/vim/.vimrc`

2.  `~/abc/.vimrc` ->  `~/dotfiles/vim/abc/.vimrc`

以我自己的 dotfiles 來說長這樣，我期望 `nvim` 底下的 `.config` 最後會變到 `~/.config`    


```sh
tree
.
├── nvim
│   └── .config
│       └── nvim
│           ├── LICENSE
│           ├── init.lua
│           ├── lazy-lock.json

│           ├── lua

```

### 如何使用

剛還沒提到用法，以下指令都在 `~/dotfiles` 下執行

想要創建連結可以使用以下指令


```sh
stow <Directory> // e.g. stow vim
```

提供給他一個或多個資料夾

他就會為該資料夾底下的所有檔案以及資料夾創建對應的軟連結  

但若是只想要為檔案的話需要加上 **--no-folding**

除非你確定連結過去的資料夾只有你會使用，不然建議使用這個 flag

刪除的話也很簡單，只需要使用 **-D** 或 **--delete** 即可

```sh 
stow -D <Directory>
```

### 將餐點送到指定的位置

看起來不錯，但如果只能創建家目錄底下的連結感覺不夠實用  

其實他預設目標並不是家目錄，而是專案的上層目錄

但我們創建的地方就在家目錄下面，所以他預設就會相對於家目錄

想要放到其他地方可以透過 **-t** 來指定，像是我想要放到 `/etc` 底下

```sh
stow -t /etc <Directory>
```

### 點點點

在管理這些檔案時前綴可能很多 . 的， ls 預設不會顯示  

也可以將前綴改為 dot-，在加上 **--dotfiles** 

stow 就會在創建連結時自動把 dot- 轉換成 .

```sh
stow --dotfiles <Directory>
```
